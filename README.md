## 接口统一规范约定：
### 协议：Http
### 风格：Restful
  - 资源名词尽量复数形式
  - PUT 用来更新资源的全部字段 PATCH 用来更新资源的部分字段
  - JSON字段采用驼峰格式
  - 失败响应格式如下:
``` javascript
{
  "errorCode": "Not Found", 
  "errorMessage":"资源未找到"
}
```
### 请求数据格式：JSON
### 响应数据格式：JSON
### 响应代码：
  - 200: 成功
  - 201: 创建成功
  - 400: 失败
  - 401: 用户认证失败或权限不够
  - 403: 资源不允许访问
  - 404: 资源不存在
  - 405: 请求方法错误
  - 500: 服务端错误
